# Marp画像サイズガイドライン

## 概要

このガイドラインは、Marpスライド（16:9フォーマット）においてHeaderとFooterを避けた表示領域に最適な画像サイズを定義します。

## スライド基本仕様

### 解像度
- **全体解像度**: 1280px × 720px（16:9）
- **フォーマット**: Widescreen

### Header/Footer領域
- **Header高さ**: 96px（上部余白8px + Header 88px）
- **Footer高さ**: 42px
- **左右padding**: 32px × 2 = 64px

### コンテンツ表示領域
- **幅**: 1280px - 64px = **1216px**
- **高さ**: 720px - 96px - 42px = **582px**
- **アスペクト比**: 約 2.09:1

```
┌─────────────────────────────────────────┐
│  Header (96px)                          │ ← ヘッダー領域
├─────────────────────────────────────────┤
│                                         │
│    コンテンツ領域                        │
│    1216px × 582px                       │ ← 画像配置可能領域
│                                         │
├─────────────────────────────────────────┤
│  Footer (42px)                          │ ← フッター領域
└─────────────────────────────────────────┘
```

## 推奨画像サイズ一覧

### 1. フルサイズ画像（最大サイズ）

コンテンツ領域いっぱいに画像を配置する場合。

- **サイズ**: **1216px × 582px**
- **用途**: スクリーンショット、図解、インフォグラフィック
- **アスペクト比**: 2.09:1

```markdown
![](image.png)
```

### 2. 安全領域サイズ（推奨）

視覚的な余白を確保し、テキストと併用しやすいサイズ。

- **サイズ**: **1100px × 550px**
- **用途**: 一般的な図解、チャート、写真
- **余白**: 上下左右に適度な余白を確保

```markdown
![center](image.png)
```

### 3. 16:9比率画像

一般的な16:9比率の画像を使用する場合。

- **横向き（高さ基準）**: **1034px × 582px**
- **用途**: 動画スクリーンショット、横長の図解
- **注意**: 幅を1216pxにすると高さが684pxとなり、はみ出します

### 4. レイアウト別推奨サイズ

#### twoColumns（2カラムレイアウト）

2カラム構成で左右に画像とテキストを配置する場合。

- **1カラムあたりの幅**: 約 576px
- **推奨画像サイズ**: **550px × 400px**
- **用途**: Before/After比較、図とテキストの並列表示

```markdown
<!-- _class: twoColumns -->

<div>

## 左側

![](image-left.png)

</div>
<div>

## 右側

![](image-right.png)

</div>
```

#### imageCenter（画像中央配置）

画像を中央に配置し、タイトルと併用する場合。

- **推奨サイズ**: **900px × 600px**
- **最大高さ**: 600px（CSS制約）
- **用途**: メイン画像の強調表示

```markdown
<!-- _class: imageCenter -->

# タイトル

![](image.png)
```

#### imageFull（全画面画像）

画像をスライド全体に表示する場合。

- **推奨サイズ**: **1280px × 720px**
- **用途**: インパクトのある画像、背景画像
- **注意**: Header/Footerも画像で覆われます

```markdown
<!-- _class: imageFull -->

![](fullscreen-image.png)
```

#### imageGrid（グリッドレイアウト）

複数の画像を2列グリッドで配置する場合。

- **推奨サイズ**: **550px × 400px** （1画像あたり）
- **用途**: 複数のスクリーンショット、ギャラリー表示

```markdown
<!-- _class: imageGrid -->

# タイトル

![](image1.png)
![](image2.png)
![](image3.png)
![](image4.png)
```

## 高解像度ディスプレイ対応

Retinaディスプレイや4Kモニターでの表示を考慮する場合、2倍解像度で作成することを推奨。

### 2倍解像度（Retina対応）

- **フルサイズ**: **2432px × 1164px**
- **安全領域**: **2200px × 1100px**
- **16:9比率**: **2068px × 1164px**

画像ファイルは2倍サイズで作成し、Markdown上では通常サイズで指定します（ブラウザが自動的にスケーリング）。

```markdown
<!-- 画像は 2200px × 1100px で作成 -->
<!-- Markdownでは通常サイズで指定 -->
![](high-res-image.png)
```

## ファイル形式推奨

### PNG
- **用途**: スクリーンショット、図解、ロゴ
- **特徴**: 透過対応、ロスレス圧縮
- **推奨**: テキストや図形を含む画像

### JPEG
- **用途**: 写真
- **特徴**: 高圧縮率、小さいファイルサイズ
- **推奨**: 写真や自然画像

### SVG
- **用途**: アイコン、ベクター図形
- **特徴**: 拡大しても劣化しない、軽量
- **推奨**: シンプルな図解、アイコン、フローチャート

```markdown
<!-- SVGは解像度に依存しない -->
![](diagram.svg)
```

## 実装例

### 基本的な画像配置

```markdown
---
marp: true
theme: github-dark
paginate: true
size: 16:9
---

# タイトル

コンテンツ領域に最適化された画像（1100px × 550px）

![center](optimized-image.png)
```

### 2カラムで画像を並べる

```markdown

<div>

## Before

![](before.png)

変更前の状態

</div>
<div>

## After

![](after.png)

変更後の状態

</div>
```

### 画像を中央に大きく配置

```markdown

# システム構成図

![](architecture.png)
```

## トラブルシューティング

### 画像がはみ出す

**原因**: 画像サイズが表示領域を超えている

**対処法**:
1. 画像を安全領域サイズ（1100px × 550px）にリサイズ
2. `![center]`を使用して中央配置
3. CSSで`max-width: 100%`を指定（すでにテーマに含まれています）

### 画像が小さすぎる

**原因**: 元画像の解像度が低い

**対処法**:
1. 高解像度版（2倍サイズ）を用意
2. ベクター形式（SVG）に変更
3. 画像を再作成

### 画像が粗い（ぼやける）

**原因**: Retinaディスプレイで表示している際に、通常解像度の画像を使用している

**対処法**:
1. 2倍解像度（2200px × 1100px）で画像を作成
2. Markdownでは通常サイズで指定
