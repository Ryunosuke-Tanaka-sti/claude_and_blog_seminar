# SVG vs HTML 比較：4フェーズワークフロー図

## 概要

4フェーズワークフロー図を作成する際、SVGとHTMLのどちらで作成するかを比較検討します。

---

## 既存の3フェーズ図解を分析

### diagram_3phase_flow.svg の特徴

**ファイル情報**:
- サイズ: 1216 × 608 px
- フォーマット: SVG
- 配色:
  - 背景: `#21262D`
  - 枠線: `#58A6FF` (青系)
  - テキスト: `#C9D1D9` (グレー系)
- デザイン: フラット、グラデーションなし

**構造**:
- 3つのフェーズボックス（計画・実装・検証）
- 矢印で接続
- 各フェーズに説明テキスト
- 日本語フォント対応

---

## 選択肢の比較

### オプション1: 手書きSVG

**メリット**:
- ✅ 既存SVGと完全に同じスタイル・品質
- ✅ ファイルサイズが小さい
- ✅ スケーラブル（拡大縮小しても劣化なし）
- ✅ テキスト選択可能
- ✅ 編集が容易（SVGエディタで修正可能）
- ✅ GitHub Darkテーマとの完全統合

**デメリット**:
- ❌ 作成に時間がかかる（手動でパス、座標を計算）
- ❌ 日本語フォント埋め込みが必要
- ❌ 複雑なレイアウトの調整が難しい

**作成方法**:
1. SVGエディタ（Figma、Illustrator、Inkscape）で作成
2. または手動でSVGコードを記述

---

### オプション2: HTML + Tailwind CSS → SVG変換

**メリット**:
- ✅ レイアウトが容易（Flexbox、Grid使用可能）
- ✅ 日本語フォントを簡単に指定
- ✅ スタイル調整が迅速
- ✅ プロトタイプを素早く作成可能

**デメリット**:
- ❌ HTMLベースなので、そのままではSVGにならない
- ❌ SVG変換が必要（追加手順）
- ❌ テキストが画像化される可能性

**作成方法**:
1. HTMLファイル作成（Tailwind CSS使用）
2. ブラウザで表示
3. SVGとしてエクスポート（手動または自動）

---

### オプション3: HTML + CSS → PNG スクリーンショット → SVG変換

**メリット**:
- ✅ 最も迅速にプロトタイプ作成
- ✅ Playwright MCPでスクリーンショット自動化
- ✅ 複雑なレイアウトも対応可能

**デメリット**:
- ❌ PNG形式（ビットマップ）なので拡大すると劣化
- ❌ ファイルサイズが大きい
- ❌ テキスト選択不可
- ❌ 既存SVGとの品質差が顕著
- ❌ **推奨されない方法**

---

### オプション4: HTML + SVG埋め込み

**メリット**:
- ✅ HTMLでレイアウト、SVGで図形描画のハイブリッド
- ✅ 日本語フォント対応が容易
- ✅ スケーラブル
- ✅ 編集が容易

**デメリット**:
- ❌ HTMLとSVGの両方の知識が必要
- ❌ やや複雑な構造

**作成方法**:
1. HTMLファイル内にSVG要素を埋め込み
2. SVG要素で図形・矢印を描画
3. HTML要素でテキスト配置

---

## 推奨方法の決定

### 推奨: **オプション1（手書きSVG）**

**理由**:
1. 既存SVG（`diagram_3phase_flow.svg`）と完全に同じ品質・スタイルを実現できる
2. スケーラブルでファイルサイズが小さい
3. 編集が容易で保守性が高い
4. GitHub Darkテーマとの完全統合

### 代替案: **オプション4（HTML + SVG埋め込み）**

HTMLで大まかなレイアウトを作成し、SVGで矢印や図形を描画する方法も検討可能。

## 決定事項

### 採用方法

**Phase 1**: HTMLプロトタイプで迅速に検証

3. 両方のアプローチの利点を活用
