# HTML/SVG作成対象の図解

## 基準

既存SVG図解（`diagram_3phase_flow.svg`）と同等のクオリティを求める図解は、HTML/SVGで作成すべきです。

### HTML/SVG作成が適切な図解の特徴

- ✅ **フロー図**: 複数のステップを矢印で結ぶ
- ✅ **概念図**: 複雑な関係性を視覚化
- ✅ **構造図**: システム構成やアーキテクチャ
- ✅ **視覚的なインパクトが必要**: プレゼンの要点となる図解

### Markdownテキストベースで十分な図解の特徴

- テキスト + ボックスの組み合わせ
- 比較表（左右対比）
- 箇条書き + 説明
- 単純なテーブル

## HTML/SVG作成の方針

### 配色規則（既存SVGを踏襲）

```css
/* 基本配色 */
background: #21262D;       /* GitHub Dark背景 */
border: #58A6FF;           /* 青系（メイン） */
text: #C9D1D9;             /* グレー系 */

/* アクセントカラー */
success: #3FB950;          /* 緑系（改善・After） */
error: #F85149;            /* 赤系（問題・Before） */
warning: #D29922;          /* 黄系（注意） */

/* 禁止事項 */
gradient: none;            /* グラデーション禁止 */
shadow: none;              /* シャドウ禁止 */
```

### サイズ規格

- **幅**: 1216px（`diagram_3phase_flow.svg`に合わせる）
- **高さ**: 適切な比率で調整（600px前後）
- **viewBox**: `0 0 1216 [height]`

### デザインガイドライン

1. **シンプル**: フラットな単色デザイン
2. **明瞭**: 矢印と文字で関係性を明確に
3. **一貫性**: 既存SVGと統一されたスタイル
4. **可読性**: テキストサイズは十分に大きく
