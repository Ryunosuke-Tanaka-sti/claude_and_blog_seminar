## 作業指示書

### 作業1: 現在のスライドをレビュー

**目的**: 現在の `ai_development_environment_setup.md` で既に実装されている図解を確認

**コマンド**:
```bash
# スライドを開いて確認
code application/marp/src/ai_development_environment_setup.md
```

**確認項目**:
- ✅ 既存SVG参照が正しいか
- ✅ テキストベース図解が仕様通りか
- ✅ 不足している図解はどれか

### 作業2: 不足図解の追加

**目的**: 仕様書で定義されているが実装されていない図解を追加

**方法**:
1. 図解リスト（1-14）と現在のスライドを比較
2. 不足している図解を特定
3. Markdownで実装を追加

### 作業3: 既存図解の改善

**目的**: 仕様に準拠していない図解を修正

**確認ポイント**:
- グラデーション使用禁止（`.highlight`除く）
- 配色が仕様通りか
- テキストベース優先になっているか

## まとめ

### 重要ポイント

1. **HTML/SVGでの新規図解作成は不要**
   - すべてMarp Markdown + CSSで実装可能
   - 既存SVG（2個）を参照するのみ

2. **作成方法の内訳**
   - Markdownテキストベース: 12個
   - 既存SVG参照: 2個
   - HTML/SVG新規作成: 0個

3. **次のアクション**
   - 現在のスライドをレビュー
   - 不足図解を特定
   - Markdownで追加実装

### HTML非推奨の理由

- グラデーション使用によりテーマとの統合が困難
- スクリーンショット方式は保守性が低い
- Markdownテキストベースで十分に表現可能
- 既存SVG図解のスタイルと整合性が取れない

---

**結論**: すべての図解はMarp Markdown内で完結し、HTML/SVGファイルの新規作成は不要です。
