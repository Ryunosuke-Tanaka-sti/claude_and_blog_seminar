# フロントエンド機能仕様書：セミナースライド一覧ページ

## 概要
Marpでビルドされた技術セミナースライド（`/slides/`配下のHTMLファイル）をトップページから参照できるように、セミナー概要を含めた一覧ページを作成します。現在、スライドは`dist/slides/`に配置されているものの、フロントエンドのトップページからリンクされておらず、直接URLを知らないとアクセスできない状態です。

## 背景・課題

### 現在の問題点
1. **アクセス性の欠如**: スライドが`/slides/`配下に存在するが、トップページからリンクがないため発見できない
2. **情報の分断**: Marpビルド結果とフロントエンドが統合されているが、導線がない
3. **スライドの概要不明**: ファイル名だけではスライドの内容や対象者が分からない

### 解決すべき課題
- トップページまたは専用ページから全てのセミナースライドにアクセス可能にする
- 各スライドのタイトル、概要、対象者などの情報を表示する
- スライドを探しやすくする（カテゴリ、タグ、検索機能など）

## 情報設計（最重要）

### 表示する情報

| 情報項目 | 必須/任意 | 優先度 | 説明 |
|---------|----------|--------|------|
| スライドタイトル | 必須 | 最重要 | セミナーの正式名称（例：「今日から使える Claude 活用で技術ブログ執筆を 3 倍速に」） |
| サムネイル画像 | 任意 | 重要 | スライドの最初のページまたは代表的なビジュアル（なければデフォルト画像） |
| 概要・説明 | 必須 | 重要 | セミナーの内容を2-3行で説明（何を学べるか、どんな内容か） |
| 対象者 | 任意 | 重要 | 想定する聴衆（例：「プログラミング初心者」「AI開発ツールに興味がある人」） |
| テーマ | 任意 | 補足 | 使用しているMarpテーマ（canyon-custom / github-dark） |
| スライド枚数 | 任意 | 補足 | 概算のスライド数（参考情報） |
| カテゴリ/タグ | 任意 | 補足 | スライドの分類（例：「Claude活用」「AI開発」「ワークフロー改善」） |
| 作成日・更新日 | 任意 | 補足 | スライドの最終更新日時 |
| スライドへのリンク | 必須 | 最重要 | `/slides/[filename].html`への直接リンク |

**優先度の定義:**
- **最重要**: タイトルとリンク（これがないとスライドにアクセスできない）
- **重要**: 概要、対象者、サムネイル（スライドを選ぶ判断材料）
- **補足**: その他のメタ情報（あると便利だが、なくても問題ない）

### 情報の構造
```
トップページ（既存）
├── 既存のWelcomeMessage
├── 既存の機能紹介カード
└── 新規セクション: セミナースライド一覧
    ├── セクションタイトル（例：「技術セミナースライド」）
    ├── セクション説明（簡単な導入文）
    └── スライドカード群（グリッド表示）
        ├── スライドカード1
        │   ├── サムネイル画像（最重要）
        │   ├── タイトル（最重要）
        │   ├── 概要（重要）
        │   ├── 対象者（重要）
        │   └── カテゴリタグ（補足）
        ├── スライドカード2
        └── ...（全11スライド）
```

## ユーザーインタラクション

### 基本操作
- **スライドカードをクリック** → スライド（`/slides/[filename].html`）を新しいタブで開く
- **カテゴリタグをクリック**（将来の拡張）→ カテゴリ別フィルタリング
- **検索ボックスに入力**（将来の拡張）→ タイトル・概要でリアルタイム検索

### ページ遷移
```
トップページ（index.html）
 ↓ スライドカードクリック
スライド詳細ページ（/slides/[filename].html）※新しいタブ
```

## データ仕様

### 入力データ

**データソース:**
- **静的データ**: スライドのメタ情報は、Astroページまたは別ファイル（JSON/YAML）で定義
- **スライドファイル**: `dist/slides/`配下のHTMLファイル（ビルド時に自動生成）

**データ構造案:**
```typescript
interface SeminarSlide {
  // 必須項目
  id: string;                  // ファイル名（拡張子なし）例: "claude_seminar_slides"
  title: string;               // スライドタイトル
  url: string;                 // スライドURL 例: "/slides/claude_seminar_slides.html"
  description: string;         // 概要（2-3行）

  // 任意項目
  thumbnail?: string;          // サムネイル画像URL（なければデフォルト画像）
  targetAudience?: string[];   // 対象者リスト 例: ["プログラミング初心者", "AI開発ツールに興味がある人"]
  theme?: 'canyon-custom' | 'github-dark';  // Marpテーマ
  slideCount?: number;         // スライド枚数
  categories?: string[];       // カテゴリタグ 例: ["Claude活用", "AI開発"]
  createdAt?: string;          // 作成日（ISO8601形式）
  updatedAt?: string;          // 更新日（ISO8601形式）
}
```

**データ定義の例:**
```typescript
const seminarSlides: SeminarSlide[] = [
  {
    id: 'claude_seminar_slides',
    title: '今日から使える Claude 活用で技術ブログ執筆を 3 倍速に',
    url: '/slides/claude_seminar_slides.html',
    description: 'Claude AIを使った技術ブログ執筆の効率化テクニックを紹介。プロンプトエンジニアリングから品質管理まで、実践的な手法を学べます。',
    targetAudience: ['技術ブロガー', 'AIツール初心者'],
    theme: 'github-dark',
    categories: ['Claude活用', 'ブログ執筆'],
  },
  // ... 他のスライド
];
```

### 表示データの変換
```
入力データ（SeminarSlide[]） → カード形式で表示

変換処理:
- thumbnail未指定 → デフォルトサムネイル画像を使用
- targetAudience配列 → カンマ区切りテキストに変換
- categories配列 → タグ形式で表示
- updatedAt（ISO8601） → "2025年11月18日"形式に変換
```

## 状態管理

### 管理すべき状態
現時点では静的データのみのため、複雑な状態管理は不要です。
将来的に検索・フィルタ機能を追加する場合は以下を検討：

- `searchQuery`: "" → 検索クエリ
- `selectedCategory`: null → 選択中のカテゴリ
- `filteredSlides`: seminarSlides → フィルタリング後のスライドリスト

## バリデーション・エラー処理

### データバリデーション
- **必須項目チェック**: `id`, `title`, `url`, `description`が存在するか
- **URL形式チェック**: `url`が`/slides/`で始まるか

### エラー表示
- **スライドデータが空** → "現在、公開中のセミナースライドはありません。"
- **データ読み込み失敗** → "スライド情報の読み込みに失敗しました。"

## 非機能要件

### パフォーマンス
- 初期ロード: 2秒以内
- サムネイル画像: 遅延ロード使用
- 画像最適化: WebP形式を優先、フォールバックにPNG/JPEG

### レスポンシブ対応
- **モバイル（〜768px）**: カードを1列で表示
- **タブレット（769px〜1024px）**: カードを2列で表示
- **デスクトップ（1025px〜）**: カードを3列で表示

### アクセシビリティ
- キーボード操作: Tabキーでカード間を移動可能
- スクリーンリーダー: 各カードに適切なaria-labelを設定
- 色のコントラスト: WCAG AA準拠

## 実装時の注意点

### 優先度
- 🔴 必須: スライド一覧の表示、各スライドへのリンク
- 🟡 推奨: サムネイル画像、カテゴリタグ、対象者情報
- 🟢 オプション: 検索機能、フィルタリング機能、ページネーション

### 実装順序
1. **データ定義**（最優先）: `seminarSlides.ts`でメタデータを定義
2. **型定義**: `SeminarSlide`インターフェースを定義
3. **カードコンポーネント**: 単一のスライドカードを実装
4. **一覧コンポーネント**: カードをグリッド表示
5. **トップページへの統合**: 既存のindex.astroに追加
6. **スタイリング**: レスポンシブ対応とアクセシビリティ確保

### 既存コードとの関係
- **既存のBaseLayoutを使用**: `@/layouts/BaseLayout.astro`
- **既存のCSSカスタムプロパティを継承**: `--color-primary`, `--color-bg-secondary`など
- **既存のfeature-cardスタイルを参考にする**: トップページの`.feature-card`と調和するデザイン

## テスト観点

### 情報表示の確認
- ✅ 全11個のスライドが表示されているか
- ✅ スライドタイトルが正しく表示されているか
- ✅ 概要文が読みやすく表示されているか
- ✅ 情報の優先度は適切か（タイトルが最も目立つか）

### インタラクションの確認
- ✅ スライドカードをクリックすると正しいURLに遷移するか
- ✅ リンクが新しいタブで開くか（`target="_blank"`）
- ✅ ホバー時に視覚的フィードバックがあるか

### エラーケースの確認
- ✅ データが空の場合に適切なメッセージが表示されるか
- ✅ サムネイル画像が存在しない場合にデフォルト画像が表示されるか

### レスポンシブの確認
- ✅ モバイル・タブレット・デスクトップで正しくグリッド表示されるか
- ✅ 各デバイスで情報の優先度が保たれているか（タイトルの視認性）

### アクセシビリティの確認
- ✅ キーボードのTabキーでカード間を移動できるか
- ✅ スクリーンリーダーで各カードの情報が読み上げられるか
- ✅ リンクに適切なaria-labelが設定されているか
