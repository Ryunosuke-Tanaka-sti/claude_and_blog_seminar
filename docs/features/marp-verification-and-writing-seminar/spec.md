# Marpスライド仕様書：AI協業開発　検証と執筆までをセットで行う環境

## 概要
AI協業開発において、検証から記事執筆までを一貫したワークフローで行う環境構築手法を解説するセミナー資料です。モノレポをベースに、4フェーズワークフロー（計画→実装→研究記録→記事化）を実践することで、知見の資産化と記事執筆の効率化を実現します。

## 目的・ゴール
このスライドで伝えたいメッセージは以下の通りです：

- **モノレポがAI協業開発の基盤**：コンテキストの一元管理でAIとの協働効率を向上
- **4フェーズワークフローで知見を資産化**：検証結果を体系的に整理し、記事化を自動化
- **研究フェーズの重要性**：実装の意思決定と学びを記録することで、記事執筆が劇的に効率化
- **RAGもどきで既存記事を活用**：トークン削減50-60%を実現し、執筆時間を短縮

## ターゲットオーディエンス
- AI協業開発を実践しているエンジニア
- 技術ブログを書きたいが執筆に時間がかかっている人
- 検証した知見を体系的に整理・資産化したい開発者
- モノレポ構成でのAI活用に興味がある人

## 伝えたいメッセージ（最重要）

### 核となるメッセージ
3つに絞ります：

1. **モノレポでAIにコンテキストを提供**：CLAUDE.md階層構造でAIが自律的にプロジェクト全体を把握
2. **4フェーズワークフローで知見を資産化**：計画→実装→研究記録→記事化のサイクルで、検証結果が自然に記事化される
3. **研究フェーズが記事執筆の鍵**：実装時の意思決定と学びを記録することで、記事執筆時間50%削減を実現

### 各メッセージの重要度
- 🔴 最重要: 4フェーズワークフローで知見を資産化（これが最も価値あるポイント）
- 🟡 重要: モノレポでAIにコンテキストを提供（基盤となる環境）
