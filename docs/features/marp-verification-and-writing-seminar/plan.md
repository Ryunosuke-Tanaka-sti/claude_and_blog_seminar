# 実装計画書：AI協業開発　検証と執筆までをセットで行う環境

## プロジェクト概要

### 目的
AI協業開発において、検証から記事執筆までを一貫したワークフローで行う環境を解説するMarpスライドを作成します。4フェーズワークフロー（計画→実装→研究記録→記事化）の価値を伝え、特に研究フェーズの重要性とRAGもどきによる記事執筆効率化を強調します。

### スコープ
- **対象範囲**: Marpスライド30-35枚の作成
- **実装場所**: `application/marp/src/verification_and_writing_workflow.md`
- **使用テーマ**: `github-dark`
- **実装時間**: 2-3時間（図解作成を含む）

## 作成するファイルの詳細

### メインファイル
**ファイルパス**: `application/marp/src/verification_and_writing_workflow.md`

**ファイルの役割**:
AI協業開発における検証から記事執筆までの一貫したワークフローを解説するMarpスライド

**主要セクション**:

   - なぜモノレポなのか（図解）
   - モノレポのメリット
   - CLAUDE.md階層構造の概要
   - CLAUDE.md階層構造の効果
   - 次のステップへの接続

3. Part 2: 4フェーズワークフロー（8-10枚）
   - セクションタイトル
   - 3フェーズから4フェーズへの進化（図解）
   - なぜフェーズを分けるのか
   - 4つのフェーズの概要
   - 実際のプロジェクト構造
   - ディレクトリ構造の図解
   - ディレクトリ分離の利点
   - フェーズ分離の実践的メリット
   - 次のステップへの接続

4. Part 3: 研究フェーズの実践（6-8枚）
   - セクションタイトル
   - 研究記録の目的
   - 研究記録に記載する内容（✅/❌形式）
   - 実際の研究記録例

   - Before/After比較
   - 次のステップへの接続

5. Part 4: 記事化フェーズ（5-6枚）
   - セクションタイトル
   - 記事化の流れ（図解）

   - 4フェーズワークフローの効果

## リスクと対策

### リスク1: 図解の作成に時間がかかる
**影響度**: 中
**対策**:
- 既存の図解（diagram_*.png/svg）を最大限再利用
- 新規作成が必要な場合は、既存の図解を参考にシンプルな構成にする
- 最悪の場合、テキストベースの説明で代替

### リスク2: スライド枚数が多くなりすぎる
**影響度**: 低
**対策**:
- 1スライド1メッセージの原則を厳守
- 詳細な説明は既存ブログ記事へのリンクで誘導
- 優先度の低い内容（🟢オプション）は削除

### リスク3: 既存スライドとの内容重複
**影響度**: 中
**対策**:
- Part 1（モノレポ）は既存スライドと同じ内容でOK（基盤の説明）
- Part 2-4は新しい視点（4フェーズ・研究・記事化）に焦点を当てる
- 既存スライドへの言及と差別化を明確にする

### リスク4: 40分の想定時間を超える
**影響度**: 中
**対策**:
- 各セクションのスライド枚数を上限枚数内に収める
- 1スライドあたり60-90秒を想定
- リハーサルで時間を測定し、調整
