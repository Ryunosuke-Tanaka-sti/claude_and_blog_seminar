## 伝えたい3つのメッセージ

### 1. 「AIが悪いのではなく、人間が悪い」

AI開発ツールが意図したコードを生成しないのは、AIの性能の問題ではなく、**人間の入力（プロンプト・設計）の問題**です。

**解決策**: 計画ドキュメント（仕様書）で意思決定を明文化する

### 2. 「意思決定は人間、実行はAI」

効果的なAI協業は、**役割分担を明確にすること**がカギです。

- **人間の役割**: 何を作るか、どのような方針で開発するか
- **AIの役割**: 決定された方針に従ってコード生成

**実践**: 4フェーズ開発ワークフロー（計画 → 実装 → 研究記録 → 記事化）

### 3. 「検証と執筆をセットで、知見を資産化する」

開発完了で終わりではなく、**検証 → 記事化をワンセット**にすることで、知見が自然に蓄積され、組織の資産になります。

**仕組み**: RAGもどき（既存記事参照）+ 文体補正システム

## ターゲット層

### 初級者向け

**ペルソナ**:
- AI開発ツールをほとんど使ったことがない開発者
- Vibe Coding（適当にプロンプトを投げる）レベルの経験のみ
- 「AI開発って本当に効率化できるの？」と懐疑的

**得たい成果**:
- ✅ AI開発の正しい使い方を学ぶ
- ✅ 開発時間を短縮する具体的な手法を知る
- ✅ すぐに実践できるワークフローを持ち帰る

### 上級者向け

**ペルソナ**:
- Claude Code / GitHub Copilot / Cursor等を既に業務で使用中
- Vibe Codingの限界を感じている
- プロンプトエンジニアリングにある程度慣れている

**得たい成果**:
- ✅ 4フェーズ開発のワークフローを理解
- ✅ CLAUDE.md階層構造の設計方法を学ぶ
- ✅ 知見を資産化する仕組みを持ち帰る
- ✅ モノレポ×AI開発環境の構築パターンを知る

## Before / After

### 定量的な効果

| 指標 | Before | After | 改善率 |
|------|--------|-------|--------|
| **開発時間** | 1週間 | 2日 | 71%短縮 |
| **記事執筆時間** | 8時間 | 4時間 | 50%削減 |
| **手戻り回数** | 平均3-4回 | 平均1回 | 75%削減 |
| **既存記事重複チェック** | 30分 | 5分 | 83%削減 |
| **CI/CDビルド時間** | 12分 | 5分 | 58%削減 |

### 開発速度

- **小規模システム**: 30分で完成（バックエンド2エンドポイント + フロントエンド1画面）
- **中規模システム**: 2日で完成（バックエンド4エンドポイント + フロントエンド複数画面）
