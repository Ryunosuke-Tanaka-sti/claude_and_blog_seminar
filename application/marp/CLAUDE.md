# Marp実装ルール（プレゼンテーションスライド）

## 概要
Marpを使用したプレゼンテーションスライドの作成ルールです。
技術ブログの内容をスライド形式で発表するための資料を作成します。

## 技術スタック
- **Marp CLI**: Markdownからスライド生成
- **フォーマット**: 16:9ワイドスクリーン
- **テーマ**: `canyon-custom.css`（明るいテーマ）、`github-dark.css`（ダークテーマ）
- **出力**: HTML形式（`dist/`ディレクトリ）

## ディレクトリ構成
```
application/marp/
├── src/                    # スライドMarkdownファイル
│   ├── *.md               # 各スライドファイル
│   ├── assets/            # 画像・SVG
│   └── html/              # インタラクティブHTML部品
├── templates/             # 再利用可能テンプレート
│   ├── base/              # ベーステンプレート
│   ├── components/        # スライドコンポーネント
│   └── examples/          # 使用例
├── theme/                 # CSSテーマファイル
└── package.json
```

## スライド作成の基本方針

### 情報設計重視
従来のスライドデザインでは「見た目」に時間を取られがちですが、
このプロジェクトでは**情報の構造化**に注力します。

**人間が注目すべきこと：**
- ✅ 伝えたいメッセージの明確さ
- ✅ 情報の順序と論理的な流れ
- ✅ 聴衆にとっての価値（何を学べるか）
- ✅ 図解すべきポイントの選定

**AIに任せること：**
- 🤖 レイアウトの調整
- 🤖 色使いとフォントサイズ
- 🤖 アニメーション効果
- 🤖 コードブロックのハイライト

## スライドファイルの構造

### YAMLフロントマター（必須）
```markdown
---
marp: true
theme: canyon-custom
paginate: true
size: 16:9
header: ''
footer: ''
---
```

### 基本的なスライド構成
1. **タイトルスライド**: タイトル、サブタイトル、著者情報
2. **アジェンダ**: 発表内容の概要
3. **本編**: 3-5セクションに分けて展開
4. **まとめ**: 重要ポイントの再確認
5. **次のアクション**: 聴衆が次に何をすべきか

## コンテンツ作成ルール

### スライド1枚あたりの情報量
- **タイトル**: 1行（30文字以内推奨）
- **本文**: 3-5個の箇条書き、または1つの図
- **補足**: 必要に応じてfooterに短い注釈

### 図解の活用
複雑な概念は図解を使用：
- **フロー図**: プロセスの流れを示す（SVG推奨）
- **比較図**: Before/Afterの違いを示す
- **構成図**: システム構成やアーキテクチャ

### コードブロック
```typescript
// 短い、本質的なコードのみ掲載
// 1スライドあたり10-15行以内
```

## テーマの使い分け

### `canyon-custom.css`（明るいテーマ）
- セミナーや発表会向け
- 明るい会場での使用
- カジュアルなトーン

### `github-dark.css`（ダークテーマ）
- 技術勉強会向け
- 暗い会場での使用
- シックなトーン

## 禁止事項
- ❌ 1スライドに情報を詰め込みすぎない
- ❌ 10pt以下の小さいフォントを使用しない
- ❌ 複雑なアニメーション効果を多用しない
- ❌ 著作権のない画像を使用しない

## 仕様書の書き方

### 良い仕様書の例
```markdown
# スライド仕様書：Claude Code入門

## 目的
Claude Codeの基本的な使い方を初心者に伝える

## ターゲット
- プログラミング初心者
- AI開発ツールに興味がある人

## 伝えたいメッセージ
1. Claude Codeは開発効率を劇的に向上させる
2. 3つの基本機能だけで使い始められる
3. 今日から実践できる

## スライド構成（25枚想定）
1. タイトルスライド
2. 自己紹介
3. アジェンダ
4. セクション1: Claude Codeとは？（5枚）
   - 概要説明
   - 従来ツールとの比較図
   - 主な機能3つ
5. セクション2: 実際に使ってみる（10枚）
   - インストール手順
   - 基本的な使い方
   - デモ画面キャプチャ
6. セクション3: 活用事例（5枚）
   - ユースケース1
   - ユースケース2
   - ユースケース3
7. まとめ（3枚）
8. 次のアクション
9. Q&A

## 図解が必要な箇所
- 従来ツールとの比較表
- Claude Codeのワークフロー図
- 活用事例のBefore/After図
```

### 避けるべき仕様書
```markdown
❌ スライド1枚目は青色の背景で...
   （デザイン詳細は書かない）

❌ タイトルは48pxのフォントサイズで...
   （フォントサイズは書かない）

❌ アニメーションはフェードインで...
   （演出の詳細は書かない）
```

## 実装の流れ

### 1. 仕様書の作成
```bash
# Marp用の仕様書テンプレートをコピー
cp docs/templates/spec-template-marp.md docs/features/marp-claude-intro.md
```

### 2. Claude Codeに実装依頼
```
以下の仕様書を読み込んでMarpスライドを作成してください：
@docs/features/marp-claude-intro.md

実装方針：
- 情報の順序と論理的な流れを最優先
- レイアウトやデザインはあなたの判断で最適化
- テーマはcanyon-customを使用
```

### 3. ビルドと確認
```bash
npm run build:marp
```

### 4. 人間のレビューポイント
- ✅ メッセージは明確に伝わるか？
- ✅ 情報の順序は論理的か？
- ✅ 図解は分かりやすいか？
- 🤖 色やフォントは気にしない（AIに任せる）

## ブログ記事からのスライド変換

ブログ記事をスライド化する際の注意点：

### 情報の取捨選択
- ブログ記事の**全ての内容**をスライドにしない
- **重要なポイント**だけを抽出
- 詳細はブログ記事へのリンクで誘導

### 構造の再設計
- ブログの段落構成をそのままスライドにしない
- スライド発表に適した**ストーリー**に再構成
- 聴衆の理解を促す順序に並び替え

### 視覚化の追加
- ブログ記事にない**図解**を追加
- 複雑な説明は**フロー図**で表現
- 比較表やチェックリストを活用

## トラブルシューティング

**問題: スライドが多すぎる**
- 対処: 1スライド1メッセージの原則を守る
- 20-30枚を目安に、それ以上は分割を検討

**問題: 情報が伝わらない**
- 対処: 仕様書の「伝えたいメッセージ」を見直す
- スライドの順序を論理的に再構成

**問題: 図解が複雑すぎる**
- 対処: 1つの図で1つの概念のみを表現
- 複数の図に分割することを検討
